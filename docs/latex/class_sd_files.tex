\doxysubsection{Detailed Description}
Se encarga de gestionar la lectura de la memoria SD. 


\begin{DoxyParams}{Parameters}
{\em file\+Name} & contiene el nombre del archivo que se va a leer. \\
\hline
{\em p\+File} & se utiliza como apuntador a alguna linea del archivo. \\
\hline
{\em file\+Type} & almacena el tipo de archivo pudiendo ser\+:
\begin{DoxyItemize}
\item 1 -\/-\/$>$ .txt
\item 2 -\/-\/$>$ .thr
\item 3 -\/-\/$>$ .bin
\item -\/1 -\/-\/$>$ ninguno de los anteriores 
\end{DoxyItemize}\\
\hline
{\em direction\+Mode} & almacena la direccion de lectura del archivo, pudiendo ser\+:
\begin{DoxyItemize}
\item 0 -\/-\/$>$ lo leera de abajo hacia arriba (en reversa)
\item 1 -\/-\/$>$ lo leera de arriba hacia abajo (ejecucion normal) 
\end{DoxyItemize}\\
\hline
{\em file} & es un objeto de tipo File para el uso de la SD. \\
\hline
{\em p\+File\+Bin} & se utiliza como apuntador a la linea de un archivo .bin. \\
\hline
{\em chars\+To\+Read} & almacena el numero de bytes que se deben leer. \\
\hline
{\em status\+File} & almacena un codigo de error que puede significar lo siguiente
\begin{DoxyItemize}
\item 0 no hubo problemas
\item 1 ya no hay mas lineas por leer en el archivo
\item -\/1 no se encontro el separador de los componentes (\textquotesingle{},\textquotesingle{} o \textquotesingle{} \textquotesingle{})
\item -\/4 no se encontro un segundo componente
\item -\/6 no es un tipo de archivo valido 
\end{DoxyItemize}\\
\hline
{\em component\+Separator} & se utiliza para identificar el caracter que separa los 2 componentes del archivo. \\
\hline
{\em line\+Separator} & se utiliza para identificar el caracter que separa a cada linea del archivo. \\
\hline
{\em data\+Buffer\+Bin} & se utiliza para almacenar los datos de la SD. \\
\hline
{\em current\+Row} & almacena una linea del archivo como texto. \\
\hline
{\em data\+Buffer} & almacena parte del archivo de la SD. \\
\hline
\end{DoxyParams}
